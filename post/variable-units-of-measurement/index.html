Why work on this

Working on this involves understanding the controlls, specially the spinedit, then finding some way to modify it so that clicking on the units “the last 2 characters of the label in the spinedit entry” shows a dropdown, from which we can select a different unit.

I find working on this as a great opportunity to get deeper insights into widget implementations.

Tip of the day dialog EasyHack

I asked Hossien for mentoring, so that I can some day work on notebookbar, and tabbed UI related implementations, and he suggested I start with How to create a new “Tip-Of-The-Day” dialog.

The blog has links to other important topics like Easyhacking: All about terminology - LibreOffice Design Team which talks about types of labels that an unocommand can have. It further links tdf#108458, which has some really informative discussion “Highly recommended for someone new to Libreoffice UI.

Storing command labels in a central place such as the officecfg module, as opposed to specifying them in the toolbar/menu xml directly, has the advantage of having a single string for each command. The disadvantage is that it makes it impossible to fine-tune labels in certain places, such as different kinds of ui elements…. tdf#108458
